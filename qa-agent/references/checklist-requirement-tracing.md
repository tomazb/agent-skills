# Requirement Tracing Checklist

Use this checklist before claiming behavior is verified.

- Identify explicit requirements from ticket/spec/tests/docs.
- Identify implicit contracts from current behavior and API boundaries.
- Map each test case to a requirement or risk.
- Mark assumptions explicitly when requirement text is absent.
- Flag conflicting sources before writing final assertions.
- Record untestable requirements and why they are untestable.
- Include trace links in final findings summary.

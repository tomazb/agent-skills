# Talk Structure Templates Reference

## Related Frameworks
- Framework 9: Job Talk Framework
- Framework 3: Empowerment Promise
- Framework 10: How to Stop

## How to Use These Templates

These are starting structures, not scripts. Adapt timing to your content. The frameworks referenced in each section tell you what technique to apply at that point.

## Template 1: Conference Keynote (45-60 min)

| Time | Section | Framework | Content |
|---|---|---|---|
| 0:00-1:00 | Opening | Empowerment Promise | Specific outcome promise. "By the end, you'll be able to..." |
| 1:00-5:00 | Vision | Job Talk | The problem someone cares about + your approach |
| 5:00-8:00 | Context | Four Heuristics (Fence) | What this IS and IS NOT. Distinguish from related ideas. |
| 8:00-12:00 | Key concept 1 | Props & Stories | Teach the hardest idea first with a physical demonstration |
| 12:00-15:00 | Cycle 1 | Four Heuristics (Cycle) | Revisit core thesis from a new angle |
| 15:00-25:00 | Proof of work | Job Talk | Steps taken, evidence, demonstrations |
| 25:00-28:00 | Audience question | Four Heuristics (Question) | Re-engage: "What would happen if...?" |
| 28:00-35:00 | Key concept 2 | Board vs. Slides | Use board for teaching, slides for data |
| 35:00-38:00 | Cycle 2 | Four Heuristics (Cycle) | Third pass at core thesis — connect to bigger picture |
| 38:00-42:00 | Near-miss | Props & Stories | Show what almost works but doesn't — sharpen boundaries |
| 42:00-45:00 | Star element | Star Framework | The surprise + slogan moment |
| 45:00-48:00 | Contributions | Job Talk + How to Stop | Mirror the opening promise. Show what was delivered. |
| 48:00-49:00 | Close | How to Stop | Joke or salute. Never "thank you." |
| 49:00+ | Q&A | — | Contributions slide stays up |

## Template 2: Technical Deep-Dive (25-30 min)

| Time | Section | Framework | Content |
|---|---|---|---|
| 0:00-1:00 | Opening | Empowerment Promise | "After this, you'll be able to [specific skill]" |
| 1:00-3:00 | Context | Four Heuristics (Fence) | What problem this solves. What it doesn't solve. |
| 3:00-8:00 | Core concept | Board vs. Slides | Board for the algorithm/architecture, slides for data |
| 8:00-10:00 | Cycle 1 | Four Heuristics (Cycle) | Revisit the core mechanism from user perspective |
| 10:00-18:00 | Deep technical | Props & Stories | Live demo, code walkthrough, or board buildup |
| 18:00-20:00 | Audience question | Four Heuristics (Question) | "What would break if we removed this component?" |
| 20:00-24:00 | Practical application | Job Talk (Proof) | Real-world results, performance numbers |
| 24:00-26:00 | Near-miss | Props & Stories | Common implementation mistake and why it fails |
| 26:00-28:00 | Contributions | How to Stop | What you can now do. Final slide stays up. |
| 28:00-29:00 | Close | How to Stop | Salute or joke |

## Template 3: Team Update / Lightning Talk (5-10 min)

| Time | Section | Framework | Content |
|---|---|---|---|
| 0:00-0:30 | Opening | Empowerment Promise | One sentence: "After this you'll know X" |
| 0:30-2:00 | Context | Four Heuristics (Fence) | What changed and what didn't |
| 2:00-6:00 | Content | Star Framework | Lead with the surprise. Use the slogan as anchor. |
| 6:00-8:00 | Practical impact | Job Talk (Contributions) | What this means for the team |
| 8:00-9:00 | Close | How to Stop | Contributions. One-line close. |

## Template 4: Thesis Defense / Oral Exam

| Phase | Framework | Content |
|---|---|---|
| Opening (5 min) | Empowerment Promise + Job Talk (Vision) | State the problem clearly. Why it matters. What's new. |
| Background (10 min) | Four Heuristics (Fence) | What exists. What's missing. Your contribution vs. prior work. |
| Method (15-20 min) | Job Talk (Proof of Work) | List the STEPS — you don't need all completed. Show methodology rigor. |
| Results (10-15 min) | Props & Stories | Demonstrate findings. Use boards for derivations. Near-miss to show robustness. |
| Contributions (5 min) | How to Stop | Explicit contributions list. What's novel. What it enables. |
| Q&A defense | — | Contributions slide stays up. For each question: acknowledge, connect to your framework, answer with evidence. |

**Oral exam-specific advice (from Winston):**
- List the steps even if not all are complete — showing the path demonstrates understanding
- Examiners want to see you think, not recite. Welcome questions as opportunities to demonstrate mastery.
- If you don't know an answer: "I haven't investigated that, but my approach would be..." is stronger than guessing

## Template 5: Internal Decision Pitch (15-20 min)

| Time | Section | Framework | Content |
|---|---|---|---|
| 0:00-0:30 | Opening | Empowerment Promise | "After this, you'll be able to decide whether we should..." |
| 0:30-2:00 | Problem | Job Talk (Vision) | The problem framed in terms the audience cares about |
| 2:00-5:00 | Fence | Four Heuristics (Fence) | What you're proposing and what you're NOT proposing |
| 5:00-10:00 | Evidence | Props & Stories | Data, demos, near-miss comparisons with alternatives |
| 10:00-12:00 | Star moment | Star Framework | Surprise that reframes the decision |
| 12:00-15:00 | Proposal | Job Talk (Contributions) | Clear ask with specific next steps |
| 15:00-16:00 | Close | How to Stop | Restate the decision and your recommendation |

## Adaptation Notes

**Stretching a template:** Add more Proof of Work sections and additional Props & Stories demonstrations. Add audience questions (one per 15-20 minutes).

**Compressing a template:** Cut to three essential phases: Promise → One Key Point (with Star elements) → Contributions. Drop cycling and audience questions. Use the Lightning Talk template.

**When time-constrained, drop frameworks in this order** (least critical first):
1. Time & Place (environment is often fixed)
2. Board vs. Slides (default to slides if unsure)
3. Near-Miss (useful but not essential)
4. Four Heuristics — Questions (works best in longer talks)
5. Never drop: Empowerment Promise, Contributions (How to Stop)

## Template Selection Guide

| Situation | Template | Key emphasis |
|---|---|---|
| Conference stage, broad audience | Keynote | Star elements, cycling, props |
| Engineering audience, specific topic | Technical Deep-Dive | Board work, live demos, near-miss |
| Standup, all-hands, demo day | Lightning Talk | Surprise-first, slogan as anchor |
| Academic defense, evaluation | Thesis Defense | Steps listed, contributions explicit |
| Seeking approval or resources | Decision Pitch | Evidence-heavy, clear ask |
